    Historical Tracking

    Configure how SteamWatch collects and stores local snapshot history for charts, 24h metrics, dynamic Xd metrics, and alert signals.

    Data Collection
    Enable trend tracking
    Saves player-count snapshots at each fetch interval. The graph, latest change, 24h stats, dynamic Xd stats, and trend alerts rely on this history.
    Fetch interval
    How often data is refreshed in the background.
    15 min
    Data retention
    Snapshots older than this are automatically purged. The long-range popup stats use this full retention window, so 10 days means 10d average and 10d gain/loss.
    7 days
    Trend Level Reference

    Trends compare the smoothed average of the last 3 snapshots to the previous 3. This gives a stable, spike-resistant alert signal. It is separate from the popup's 24h and dynamic Xd summary tiles.

    Level Range Interpretation
    🚀🚀 Explosion > +50%Viral moment / major launch
    🚀 Strong Rise +20% → +50%Big patch, content drop
    📈 Rising +5% → +20%Healthy organic growth
    ➡️ Stable -5% → +5%Normal daily fluctuation
    📉 Declining -20% → -5%Losing momentum
    ⬇️ Strong Drop-50% → -20%Significant player loss
    💀 Crash < -50%Game on life support

    Notifications

    Global defaults apply to all games unless overridden individually in the Games section.

    Global Switches
    Enable notifications
    Master switch. When off, no alerts are sent for any game.
    Spike detection
    Alert when a single-interval change exceeds ±40% — indicates a patch, hotfix, or viral event.
    Default Thresholds
    📈 Rise alert
    Notify when the smoothed trend exceeds this positive percentage.
    +30%
    📉 Drop alert
    Notify when the trend falls below this negative percentage.
    -25%
    💀 Crash alert
    High-priority notification for catastrophic player drops.
    -50%
    Quiet Hours

    Suppress all notifications during a defined time window. The cooldown timer is not consumed during quiet hours, so the first eligible alert after the window ends will still fire normally.

    Enable quiet hours
    No notifications will be sent during the window below, on the selected days.

    Export Data

    Download all your tracked player-count history as a file. Useful for external analysis, backups, or sharing.

    Snapshot History

    Exports every player-count snapshot saved for all tracked games. Each row includes the game name, appid, timestamp and concurrent player count. Only data stored locally in your browser is included — nothing is fetched from the internet during export.

    { }
    JSON
    Structured array. Best for developers, scripts, or importing into other tools.
    CSV
    CSV
    Flat spreadsheet format. Open directly in Excel, Google Sheets, or Grafana.
    What gets exported
    Field Type Description
    appid string Steam application ID
    name string Game name at time of tracking
    date string ISO 8601 timestamp (UTC)
    current number Concurrent players at sample time
    ts number Unix timestamp in milliseconds
